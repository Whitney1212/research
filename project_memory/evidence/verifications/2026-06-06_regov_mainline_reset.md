# 2026-06-06 REgov 主线重设

## 记录目的

根据用户在 2026-06-06 当前对话中的明确要求，重设整个 REgov 项目的总主线：后续不再把 `09:00` CO2 次高峰事件机制归因作为项目主线，而是把它降为复杂地形通量计量修正框架下的复现与支撑证据分支。 [来源: 用户当前对话 2026-06-06]

## 新的总目标

- REgov 的总目标应明确写为：**复杂地形通量计量修正**。 [来源: 用户当前对话 2026-06-06]
- 最集中的主问题应表述为：**复杂地形下 EC 通量偏差是否由可观测的储存—平流—局地环流状态系统控制，并能否通过 CO2 廓线、多站风场和移动切面观测进行诊断、分级和修正？** [来源: 用户当前对话 2026-06-06]
- 更短的项目题目式表述可写为：**复杂地形 EC 通量偏差的状态依赖性、机制来源与可观测修正框架。** [来源: 用户当前对话 2026-06-06]

## 应作为主线的五个议题

1. **EC 通量在复杂地形中何时失效、何时仍可用**：目标不是泛泛否定 EC，而是给出可判别的适用条件。 [来源: 用户当前对话 2026-06-06]
2. **储存项如何改变 EC 对碳交换的解释**：优先推进 `EC + storage`，并量化其对夜间、晨间以及更长时间尺度碳收支的影响。 [来源: 用户当前对话 2026-06-06]
3. **平流、通风和局地环流如何造成 EC 与真实交换解耦**：关键是判断 `EC + storage` 后剩余残差是否能由风向转变、通风、外来气团输入或横谷再分配解释。 [来源: 用户当前对话 2026-06-06]
4. **移动平台切面能否提供单塔无法获得的空间约束**：`FL` 不应被包装成第三个固定通量站，而应作为平流和空间异质性的约束层。 [来源: 用户当前对话 2026-06-06]
5. **能否形成可迁移的复杂地形通量状态分类**：最终应把观测划分为 `EC 可信型`、`储存主导型`、`外来输入型`、`通风带走型`、`横谷再分配型` 和 `方法高不确定型` 等状态，并为每类状态给出对应修正策略。 [来源: 用户当前对话 2026-06-06]

## 对现有事件机制分析的定位调整

- `09:00` CO2 次高峰、事件 lead-lag、竞争性假设和 `FL` 空间形态分类仍然有价值，但它们应调整为**复现和支撑证据层**，用于说明复杂地形下储存、平流、通风和局地环流何时干扰 EC 解释。 [来源: 用户当前对话 2026-06-06]
- 因此，事件机制分析的角色不再是“项目主线本身”，而是服务于“复杂地形通量状态分类与修正框架”的一个具体案例分支。 [来源: 用户当前对话 2026-06-06] [推断：基于用户给出的总目标与五议题结构整理]
