# 2026-06-17 晨间 peak 双主线规划摘要

## 来源

- 这份记录整理自用户在当前对话中提供的 2026-06-17 规划说明，主题是把晨间 peak 相关分析从原来的复杂地形通量修正支撑案例中独立出来，形成与长期碳收支主线并行的工作流。 [来源: 用户当前对话 2026-06-17]

## 本次新增信息

- 后续项目目标调整为双主线：第一条主线继续面向复杂地形下长期碳收支问题，第二条主线独立研究长期稳定晨间 CO2 事件的边界层转换、空间传播与局地再分配机制。 [来源: 用户当前对话 2026-06-17]
- 晨间 peak 工作流的预期数据组合包括两年双固定塔连续数据、`20-30` 个三站独立事件日、移动站固定在谷中央空中的补充观测，以及 AP 廓线。 [来源: 用户当前对话 2026-06-17]
- 两年双固定塔数据不应只混合做平均日变化，而应采用“第一年建立规则、第二年独立验证”的设计；事件检测规则应在第一年冻结，再应用到第二年。 [来源: 用户当前对话 2026-06-17]
- 三站独立事件日适合机制验证，统计独立样本应定义为“事件日”，不应把 `1 min` 或 `30 min` 数据点当成独立样本；`20-30` 个事件更适合预定义机制判据、事件级回归和事件级 bootstrap，而不是复杂机器学习。 [来源: 用户当前对话 2026-06-17]
- 固定谷中央上空观测建议实际安排 `16-20` 个早晨，争取得到 `12-15` 个有效事件；若三站联合观测总量只有 `20-30` 天，建议 `12-15` 天固定谷中央上空、`8-10` 天保留移动切面。 [来源: 用户当前对话 2026-06-17]
- AP 廓线可以作为机制证据，但当前应使用廓线梯度指数、柱浓度异常代理量和柱异常变化率代理，不能称为正式 `storage flux`，也不能直接写成 `F_EC + F_storage` 的碳收支闭合。 [来源: 用户当前对话 2026-06-17]

## 需要提升为锚点的内容

- 晨间 peak 应独立为项目工作流，问题表述聚焦“长期稳定晨间 CO2 事件的边界层转换、空间传播与局地再分配机制”，不再只是复杂地形通量修正框架中的单个案例。 [来源: 用户当前对话 2026-06-17]
- AP 廓线的柱异常变化量只能称为 `column anomaly tendency proxy`、柱浓度异常变化代理或廓线库存变化指数；在缺少标准 storage flux 单位、跨系统校准和控制体定义之前，不应称为正式 storage flux。 [来源: 用户当前对话 2026-06-17]

## 仍待确认

- `20-30` 个三站独立事件日是否覆盖弱风晴空、风向转变或增风、较强通风三类状态，仍需要先做事件日气象条件分布核查。 [来源: 用户当前对话 2026-06-17]
