# 下一步主线任务

## 当前总目标

2026-06-06 后，REgov 的主线应明确收束为：**复杂地形通量计量修正**。最集中的项目问题不是单独解释某一个 CO2 事件，而是判断复杂地形下 EC 通量偏差是否由可观测的储存—平流—局地环流状态系统控制，并能否通过 CO2 廓线、多站风场和移动切面观测进行诊断、分级和修正。 [来源: 用户当前对话 2026-06-06] [已核验: project_memory/evidence/verifications/2026-06-06_regov_mainline_reset.md]

这条主线要求把现有 `EA/EC`、CO2 廓线、三站水平风、FL moving-transect、rotation 敏感性和 raw-w 诊断结果组织成一个**状态依赖的修正框架**，而不是继续把事件归因本身当作最终目标。 [来源: 用户当前对话 2026-06-06] [推断：基于用户给出的总问题和五议题整理]

## 五个主线议题

1. **EC 何时失效、何时仍可用**：需要给出复杂地形下可判别的适用条件，而不是笼统说 EC 不适用。 [来源: 用户当前对话 2026-06-06]
2. **storage 如何改变 EC 对碳交换的解释**：优先推进 `CVT` 与 `MT` 局地柱 storage tendency，并评估 `EC + storage` 对夜间、晨间及更长时间尺度碳收支的影响。 [来源: 用户当前对话 2026-06-06] [已核验: regov_memory/00_shared_research_context.md]
3. **平流、通风和局地环流如何造成 EC 与真实交换解耦**：重点不是直接把 raw-w 并入最终通量，而是判断 `EC + storage` 之后剩余偏差何时更像外来输入、通风带走或横谷再分配。 [来源: 用户当前对话 2026-06-06]
4. **FL 是否提供单塔无法获得的空间约束**：`FL` 应固定定位为平流和空间异质性的证据层，而不是第三个平均通量点。 [来源: 用户当前对话 2026-06-06]
5. **能否形成可迁移的状态分类**：目标状态至少包括 `EC 可信型`、`储存主导型`、`外来输入型`、`通风带走型`、`横谷再分配型` 和 `方法高不确定型`。 [来源: 用户当前对话 2026-06-06]

## 当前优先任务

1. **先把 EC 可信性问题结构化**。基于现有 `EA/EC`、三站风场、弱湍流/稳定度信息和 rotation 敏感性结果，整理“哪些条件下 EC 仍可作为交换参照，哪些条件下只能当作受复杂地形污染的部分证据”。这里优先产出的是状态判据，不是新的通量图。 [来源: 用户当前对话 2026-06-06] [已核验: project_memory/evidence/verifications/2026-06-01_common_rotation_sensitivity_analysis.md] [已核验: project_memory/evidence/verifications/2026-06-02_common_rotation_w_sigma_wind_stability.md]
2. **把 storage 提升为第一优先修正项**。两套廓线系统最高层已等于各自 EC 高度，因此下一步应优先做 `CVT` 和 `MT` 的局地柱 CO2 storage tendency，并明确它能解释哪些夜间/晨间 CO2 变化，不能解释哪些跨站或控制体问题。 [来源: 用户当前对话 2026-06-06] [已核验: regov_memory/00_shared_research_context.md]
3. **把平流/通风/局地环流放到 residual 解释层**。在 `EC + storage` 框架之外，再用 corrected wind sector、多站同步性、raw-w 诊断边界和 FL 形态去判断剩余偏差更像外来输入、通风带走还是横谷再分配。 [来源: 用户当前对话 2026-06-06] [推断：基于五议题第 3、4 条整理]
4. **固定 FL 的角色**。FL 后续默认服务于“单塔看不到的空间约束”：区分整区同步、单侧输入、谷底增强、横谷再分配或通风带走；不再把 FL 结果包装成独立平均通量站。 [来源: 用户当前对话 2026-06-06] [已核验: project_memory/evidence/verifications/2026-06-03_fl_moving_transect_anomaly_transport_feasibility.md]
5. **搭建状态分类表**。把现有结果整理成“状态 -> 诊断信号 -> 修正策略 -> 方法风险”的骨架，而不是继续堆叠单个现象解释。 [来源: 用户当前对话 2026-06-06]
6. **把事件机制分析降为复现分支**。`09:00` CO2 次高峰、lead-lag、机制候选排序和 FL 事件形态分类仍然保留，但它们的角色调整为：复现复杂地形下储存、平流、通风和局地环流如何干扰 EC 解释，而不是承担整个项目主叙事。 [来源: 用户当前对话 2026-06-06] [已核验: project_memory/evidence/verifications/2026-06-06_regov_mainline_reset.md]

## 当前最小步

下一最小步应先输出一版**复杂地形通量状态框架草表**，至少包括：

- `EC 可信型`
- `储存主导型`
- `外来输入型`
- `通风带走型`
- `横谷再分配型`
- `方法高不确定型`

每个状态至少写清四件事：`判别信号`、`需要的数据`、`推荐修正或解释策略`、`当前已有证据与缺口`。事件机制表暂时只作为上述状态的案例支撑，不再单独决定主线优先级。 [来源: 用户当前对话 2026-06-06] [推断：基于用户给出的状态表和主问题整理]

## 验证完成标准

- 如果能明确给出“什么情况下 EC 仍可解释、什么情况下必须加 storage、什么情况下应加平流/空间风险标记”，就说明主线已经从现象整理推进到计量框架。 [来源: 用户当前对话 2026-06-06] [推断：基于五议题结构整理]
- 如果能把 `FL` 的空间形态证据稳定接入 `EC + storage` 剩余偏差解释层，而不是把它单独写成第三通量站，就说明移动切面角色已经校正到位。 [来源: 用户当前对话 2026-06-06]
- 如果最终能形成“状态分类 + 修正策略 + 方法不确定性”三联表，项目结果就从本站案例提升为可迁移的复杂地形通量方法框架。 [来源: 用户当前对话 2026-06-06]
