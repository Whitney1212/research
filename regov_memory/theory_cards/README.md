# Theory Cards 说明

## 理论卡是什么意思

理论卡是一个小型、可追溯的理论备忘录。它不是论文综述，也不是最终结论，而是给 REgov 在讨论机制或选择方法前调用的“概念边界卡片”。

一张理论卡通常回答：

- 这个概念或方法到底定义了什么。
- 它依赖哪些假设。
- 关键公式或变量是什么。
- 在当前研究区可以观察到什么证据。
- 它不能证明什么，容易被误用在哪里。
- 本地项目记忆、代码、数据或文献来源是什么。

## 为什么需要理论卡

当前研究问题很容易把几个层次混在一起：观测现象、诊断量、计算代理、正式通量、物理机制和论文表述。理论卡的作用是把这些层次提前分开，降低后续讨论中的幻觉和过度归因。

例如 raw `w` 总输送是真实计算结果，`w_mean` 主导也是真实结果，但它不能直接等同于生态系统 CO2 交换强度。这样的边界就适合写成理论卡或方法卡，后续每次讨论相关问题时直接调用。

## 建议优先建立的卡片

- `ec_assumptions.md`：EC 通量成立条件、平稳性、均匀性和复杂地形问题。 [已建: 2026-05-28]
- `storage_flux.md`：CO2 储存项、廓线积分、夜间积累和日出释放。 [已建: 2026-05-28]
- `vertical_advection.md`：平均垂直输送、raw `w`、坐标旋转和地形垂直风的边界。 [已建: 2026-05-28]
- `horizontal_advection.md`：水平平流、站点差异、谷轴/切面输送。
- `coordinate_rotation.md`：double rotation、planar fit、sonic 坐标和经验 `w_mean ~ u + v` 诊断的区别。 [已建: 2026-05-28]
- `wpl_density_units.md`：WPL、空气密度、摩尔分数和通量单位换算。
- `ea_rea_conditional_sampling.md`：EA/REA/条件气团分组之间的定义差异。
- `lee_method.md`：Lee 相关方法在当前项目中的可用边界和不应直接升级的部分。

## 卡片模板

```markdown
# 理论卡：名称

## 定义

## 适用前提

## 核心公式或变量

## 当前项目可用证据

## 不能说明什么

## 与最终通量方案的关系

## 来源
```
