# 理论卡：EC 通量成立条件

## 定义

EC 通量在当前项目中先理解为某一观测高度上的湍流协方差项：

\[
F_{EC} = \overline{w'c'}
\]

其中 \(w'\) 是平均风分解后的垂直速度脉动，\(c'\) 是 CO2 或其他标量的浓度脉动。它直接度量的是观测点处的湍流垂直交换，不自动等同于完整生态系统源汇，也不自动包含储存、水平平流和平均垂直输送。

## 适用前提

- 统计窗口内湍流结构近似平稳。
- 观测 footprint 和源区在窗口内不要发生无法解释的剧烈变化。
- 下垫面水平非均匀性、坡地流、排泄流和局地环流对通量散度的影响可以忽略，或被单独诊断。
- 坐标口径明确：\(w'\) 是在哪个坐标系中定义的，是否经过 double rotation、planar fit 或其他旋转。
- 标量单位和密度修正口径明确：当前项目早期结果尚未默认做 WPL、频率修正和密度换算。

## 核心公式或变量

- 湍流通量：\(\overline{w'c'}\)。
- 平均/脉动分解：\(w = \bar w + w'\)，\(c = \bar c + c'\)。
- 总预算中还可能出现：储存项、水平平流项、平均垂直平流项、源汇项。

## 当前项目可用证据

- 当前 EA 实现使用 \(w'=w-\bar w\)，`F_EA_general` 与 `F_EC_cov` 数值上几乎等价，因此这一分支更接近 EC 协方差通量的条件积分写法。 [已核验: `regov_memory/00_shared_research_context.md`]
- 当前主线曾明确不做坐标旋转、WPL、频率修正和空气密度或摩尔密度换算，因此这些结果应先放在“当前坐标和当前单位下的运动学通量”层面解释。 [已核验: `regov_memory/00_shared_research_context.md`]
- `D:\00 博士阶段\博一\05 Project\ecpreproc` 是可调用的本地 EC 复现包，后续可以用于逐步引入常规 EC 处理步骤并与 EddyPro 对照。 [来源: 用户当前对话 2026-05-28]

## 不能说明什么

- 单独的 \(\overline{w'c'}\) 不能证明复杂地形下 NEE 已经完整闭合。
- 在夜间稳定层结、排泄流、局地环流或强水平浓度梯度条件下，EC 低值不一定等于生态呼吸弱。
- 未经明确坐标旋转的 \(w'\) 不能直接解释为严格地理垂直方向上的湍流交换。
- 未做 WPL、密度和单位换算时，不应把结果直接与常规 \(\mu mol\,m^{-2}\,s^{-1}\) 口径的 CO2 通量混用。

## 与最终通量方案的关系

EC 是最终通量计量方案的核心基准项，但在本研究区不能单独承担全部解释。更稳的路径是先建立一个可复现的 EC 基础版本，再逐步判断储存项、平流项、坐标旋转、谱修正、WPL 和数据筛选是否需要进入正式方案。

## 下一步验证

- 用 `ecpreproc` 明确一版“标准 EC 基线”：QC、lag、去趋势、坐标旋转、WPL、谱修正分别开启或关闭。
- 对同一批窗口输出处理状态表，避免后续把不同处理层级的结果混在一起。
- 在强稳定、低风速、明显 CO2 次高峰和强 raw `w_mean` 时段单独检查 EC 代表性。
