# 理论卡：CO2 储存项

## 定义

CO2 储存项描述观测高度以下空气柱中 CO2 浓度随时间的变化。它回答的问题是：在某一时间窗口内，空气柱本身是否正在积累或释放 CO2。

简化写法为：

\[
F_{storage} = \int_{0}^{z_r} \frac{\partial \bar c(z,t)}{\partial t}\,dz
\]

如果要换成常规 CO2 通量单位，需要进一步处理空气摩尔密度、干/湿空气口径和单位换算。

## 适用前提

- 需要有足够层位的 CO2 廓线，至少能代表观测高度以下的垂直浓度结构。
- 各层位高度、采样时间、阀号、缺测和质量控制必须清楚。
- 时间差分窗口必须和 EC/气象窗口口径匹配，尤其要区分 start-label 和 end-label。
- 储存项只处理“空气柱内浓度变化”，不自动解释这些 CO2 从哪里来、向哪里去。

## 核心公式或变量

- \(\bar c(z,t)\)：高度 \(z\) 处的平均 CO2 浓度。
- \(z_r\)：EC 或目标参考高度。
- \(\partial \bar c / \partial t\)：浓度随时间变化率。
- 层化近似：用 AP200 或其他廓线层位对积分做分层加权。

## 当前项目可用证据

- 谷底中心塔有 AP200 五层 CO2 廓线系统，谷底塔附近冠层高度约 `20 m`，因此“冠层以上/以下”的层位边界可以被明确写入储存项解释。 [已核验: `regov_memory/00_shared_research_context.md`]
- 用户明确补充，两套廓线系统的最高层分别是对应 EC 系统的观测高度。因此对 `CVT` 与 `MT` 而言，AP/profile 廓线在层位覆盖上足以分别计算到各自 \(z_r\) 的局地柱 storage tendency，不需要为最高廓线层到 EC 高度之间另做外推。 [来源: 用户当前对话 2026-05-29]
- `04 Lee` 中夜间 CO2 积累不能直接写成上午 `09:00-11:00` 次高峰强度的充分解释；它更像 CO2 库，是否形成次高峰还取决于上午输送、释放窗口、通风清除能力和垂直结构变化。 [已核验: `regov_memory/00_shared_research_context.md`]
- `CVT 2025-03-23` AP 廓线第七层 `valve_number = 7/c32` 在 `2025-03-23 17:53:30` 之后必须筛除；该日黄昏阶段不能继续解释完整廓线结构。 [已核验: `regov_memory/00_shared_research_context.md`]

## 不能说明什么

- 储存项为正或为负，不等于生态系统源汇本身增强或减弱。
- 夜间积累量大，不必然导致上午次高峰强。
- 储存项不能替代水平平流或垂直平流；它只说明空气柱浓度库存变化。
- 如果廓线层位不足或层位冲突未解决，储存项只能作为诊断量，不能直接进入最终通量闭合。
- 两套廓线最高层等于各自 EC 高度，只解决站点内垂直积分高度问题；它不自动解决跨系统绝对浓度可比性、水平浓度梯度、空间代表性和完整峡谷控制体 storage 问题。 [来源: 用户当前对话 2026-05-29] [推断：基于 storage 适用前提整理]

## 与最终通量方案的关系

储存项是复杂地形通量计量中必须优先评估的候选项。由于当前两套廓线最高层分别等于对应 EC 观测高度，`CVT` 与 `MT` 的局地柱 storage 可以作为比水平平流更优先的定量修正项推进。它可能把 EC 观测高度处的湍流通量与冠层/谷底空气柱内 CO2 变化连接起来，但它不能单独解决平流和局地环流问题，也不能在缺乏空间权重和跨系统校准时直接代表完整峡谷控制体 storage。 [来源: 用户当前对话 2026-05-29] [推断：基于当前数据条件整理]
