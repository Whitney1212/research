# REgov 研究问题地图

## 定位

这份文件记录 REgov 的最高层研究问题和子问题分层。它不替代具体 workstream 的 `project_memory`，而是帮助后续每一条分析线回到同一个总目标下。 [来源: 用户当前对话 2026-05-28]

## 总问题

复杂地形下 EC 通量偏差是否由可观测的储存—平流—局地环流状态系统控制，并能否通过 CO2 廓线、多站风场和移动切面观测进行诊断、分级和修正？ [来源: 用户当前对话 2026-06-06]

更短的项目标题式表述可以写为：**复杂地形 EC 通量偏差的状态依赖性、机制来源与可观测修正框架。** [来源: 用户当前对话 2026-06-06]

这里的“修正”不是只指往现有 EC 结果上机械加一项，而是同时包括：EC 何时仍可解释，何时必须加入 storage，何时应标记平流或空间异质性风险，何时只能给出高不确定性状态。 [来源: 用户当前对话 2026-06-06] [推断：基于用户给出的五议题和状态表整理]

## 五个核心议题

1. **EC 通量在复杂地形中何时失效、何时仍可用**  
   目标是给出复杂地形下的可判别适用条件，而不是只说 EC 不适合山地。 [来源: 用户当前对话 2026-06-06]

2. **储存项如何改变 EC 对碳交换的解释**  
   这是最优先的修正分支，因为它最接近标准通量收支框架，也最容易形成可外推的方法。 [来源: 用户当前对话 2026-06-06]

3. **平流、通风和局地环流如何造成 EC 与真实交换解耦**  
   关键问题是 `EC + storage` 之后的残差何时更像风向转变、外来输入、通风带走或横谷再分配。 [来源: 用户当前对话 2026-06-06]

4. **移动平台切面能否提供单塔无法获得的空间约束**  
   `FL` 的独特价值是判断 CO2 异常究竟更像整区同步、单侧输入、谷底增强还是横谷再分配。 [来源: 用户当前对话 2026-06-06]

5. **能否形成可迁移的复杂地形通量状态分类**  
   最终目标是形成 `EC 可信型`、`储存主导型`、`外来输入型`、`通风带走型`、`横谷再分配型` 和 `方法高不确定型` 等状态及对应修正策略。 [来源: 用户当前对话 2026-06-06]

## 当前已知牵引问题

- 当前项目不缺少事件图或现象描述，真正缺的是“状态判据 + 修正策略 + 方法风险”的统一组织框架。 [来源: 用户当前对话 2026-06-06] [推断：基于本轮主线重设整理]
- raw `w` CO2 总输送几乎由 `F_mean` 和 `w_mean` 控制，因此它目前更像平均垂直运动诊断量，不能直接升级为生态 CO2 交换强度。 [已核验: regov_memory/00_shared_research_context.md]
- 根据用户基于既往处理经验和研究区背景的判断，`F_EC` 对坐标旋转方法较为敏感。因此“基础通量计算”这一子问题不能只问某一个 EC 结果是多少，而要问不同旋转口径下哪些通量结论稳定、哪些结论应被标记为旋转敏感，以及这种敏感性是否限制最终通量计量方案。 [来源: 用户当前对话 2026-05-29] [已核验: regov_memory/00_shared_research_context.md]
- 两套廓线系统最高层分别等于对应 EC 观测高度，因此 storage 子问题可以优先推进为 `CVT` 与 `MT` 各自的局地柱 storage tendency 计算；但它仍不能自动解决完整峡谷控制体 storage、水平梯度或跨站绝对差值问题。 [来源: 用户当前对话 2026-05-29] [已核验: regov_memory/00_shared_research_context.md]
- `CVT`、`MT`、`FL` 的站点和轨道几何已经成为解释局地环流、切面输送和垂直运动差异的硬背景。 [已核验: regov_memory/00_shared_research_context.md]
- `04 Lee` 中上午 `09:00-11:00` CO2 次高峰、结构切换、夜间积累和移动平台空间形态仍是重要机制线索，但它们现在应降为服务复杂地形修正框架的复现和案例支线，而不是整个 REgov 的总主线。 [来源: 用户当前对话 2026-06-06] [已核验: project_memory/evidence/verifications/2026-06-06_regov_mainline_reset.md]
- `ecpreproc` 可以作为灵活通量计算的本地工具基础，使后续不必只依赖 EddyPro 黑箱结果。 [来源: 用户当前对话 2026-05-28]

## 每个新问题进入 REgov 前的判断

每个新分析角度都先回答四个问题：

- 它服务于最终通量计量方案的哪一部分。
- 它处理的是正式通量、修正项、诊断量、机制解释，还是数据质量问题。
- 它需要哪些已存在数据和哪些还未确认的数据。
- 它的结论如果成立，会改变最终通量结果、改变解释边界，还是只改变后续优先级。
